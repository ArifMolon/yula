# YULA Design Direction — Pointer (Faz 1 Desktop Alpha)

## Status

This is a **pointer**, not a full design document. It records the design-vision items the product owner defined during issue #3 grilling that are deliberately kept out of ADR-0013 (ADRs are single-decision records; ADR-0013 authorizes only the StyleX styling technology). The full design-direction document is to be drafted by a later Faz 1 task.

## Scope: Faz 1 — Desktop Alpha

The following form the product design vision for the Faz 1 Desktop Alpha and are owned by the Interaction (and Agent Studio UI surface) context unless a later design split says otherwise:

- **Japon minimalism** — the overarching aesthetic stance.
- **Mockups** — concrete screen designs to be produced in Faz 1.
- **Design System** — the token + component system built on top of ADR-0013's StyleX tokens.
- **Agent Characters** — the visual representation of agents.
- **Skill Icons** — the visual representation of skills.
- **Visual approval loop** — the UI surface for the Review Inbox / ApprovalRequest interaction.

## Relation to ADR-0013

ADR-0013 authorizes the styling technology (StyleX) and the single token source. Everything above is product design work that depends on, but is not part of, that one ADR. When the full design-direction document is drafted, this pointer is replaced by it (or it grows into it).

## Non-goals

This pointer does not ratify any concrete mockup, character, or icon. It only names the work so it is not lost between #3 and the Faz 1 task that drafts it.
